2. High-Level Purpose
==================================================

This is a lightweight LMS / PJBL learning platform for climate statistics with:
- student portal
- combined Teacher/Admin portal
- Supabase-backed submissions
- lesson draft persistence
- lesson scoring + lesson feedback
- analytics, exports, and class records

4. Current Active Source Of Truth
==================================================

Authoritative tables:
- public.users
- public.classes
- public.class_students
- public.responses
- public.feedback
- public.student_state

What each table does now:
- users
  - app identities, usernames, roles, hashed student passwords
- classes
  - class records / section ownership
- class_students
  - enrollment mapping
- responses
  - final submissions for:
    - pre
    - lesson1
    - lesson2
    - lesson3
    - post
  - teacher lesson scores
- feedback
  - lesson feedback records
- student_state
  - lesson draft / in-progress state snapshots

Important:
- student_progress is legacy, not the live source of truth
- lessons table is not the active lesson-output store
- browser localStorage / IndexedDB still exist as fallback cache only


==================================================
5. Actual Current Behavior Rules
==================================================

This section overrides older docs if they conflict.

Student progression:
- Pre-Assessment completion unlocks Lesson 1
- Lesson 1 completion unlocks Lesson 2
- Lesson 2 completion unlocks Lesson 3
- Lesson 3 completion unlocks Post-Assessment

Important:
- progression is completion/submission-based
- progression no longer depends on teacher feedback acknowledgment

Assessment feedback:
- Pre-Assessment feedback flow was removed from active student/admin UI
- Post-Assessment feedback flow was removed from active student/admin UI
- lessons still keep feedback support

Student lesson status:
- cards can show `Not started`, `In progress`, or `Completed`
- local progress fallback is used to avoid false `Not started` resets

Lesson 1 Activity 4:
- student no longer needs teacher feedback to revise/finalize the research question
- scoped Activity 4 feedback can still exist on the admin side

Performance summary:
- reflects assessment scores and lesson scores
- still uses timed refresh while the page is open
